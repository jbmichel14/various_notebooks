First, we create a dict mapping movieId'd to their set of genres

Then, we define the distance method that uses the Jaccard index and the k_medoids algorithm, applied to a dict X:

In [11]:
with open("most-rated.pickle", "rb") as f:
    most_rated = pickle.load(f, encoding="utf-8")
In [12]:
id_title_dict = dict(most_rated)
In [13]:
# We create a dict of the most rated movies id's mapped to their genre, which is the input of our algorithm
X = {}
for x in most_rated:
    X.update({x[0]: id_genres_dict[x[0]]})
In [14]:
k = 2
In [19]:
C1, m1 = k_medoids(X,k,d_J)
print(m1)

[(2959, ['Action', 'Crime', 'Drama', 'Thriller']), (2858, ['Comedy', 'Drama'])]
In [18]:
C2, m2 = k_medoids(X,k,d_J)
print(m2)

[(2858, ['Comedy', 'Drama']), (480, ['Action', 'Adventure', 'Sci-Fi', 'Thriller'])]

The medoids' average dissimilarity to the other members of their corresponding cluster is minimal. As the most-rated file contains only 1000 movies and k-medoids involves randomness, we do not always get the same result but the general trend for the medoids is as follows:

  • on one side 'Comedy', 'Drama', 'Romance'
  • on the other side: 'Action', 'Adventure', 'Sci-Fi', 'Thriller'

which makes a lot of sense.
